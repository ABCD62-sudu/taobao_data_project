# 淘宝用户行为路径分析与 A/B 假设检验

## 一、项目背景与目标

基于阿里天池亿级用户行为日志，构建用户转化漏斗、完成行为路径分析，并运用假设检验对关键业务假设进行严谨的统计验证，输出经统计证据支撑的运营优化建议。

---

## 二、数据集说明

| 项目 | 说明 |
|------|------|
| 数据来源 | 阿里天池公开数据集 UserBehavior.csv |
| 原始规模 | 约 1 亿条用户行为记录 |
| 数据规模 | 由于数据量过大，本项目只提取了前200万条数据|
| 时间跨度 | 2017-11-25 至 2017-12-03（约 9 天）|

**字段说明：**

| 字段 | 含义 |
|------|------|
| `user_id` | 用户 ID |
| `item_id` | 商品 ID |
| `category_id` | 商品类目 ID |
| `behavior_type` | 行为类型：`pv`(浏览) / `fav`(收藏) / `cart`(加购) / `buy`(购买) |
| `timestamp` | 行为发生时间戳（Unix 秒级，**UTC 时区**）|

---

## 三、技术栈

- **语言**：Python 3
- **数据处理**：Pandas、NumPy
- **可视化**：Matplotlib、Seaborn
- **统计检验**：SciPy（卡方检验、T 检验、Shapiro 正态性检验、Mann-Whitney U 检验、Bootstrap 自助法）

---

## 四、分析流程

```
描述性分析（业务全景） → 转化漏斗与行为路径
        → 用户分层 → 假设检验（A/B 验证） → 结论与建议
```

### 1、描述性分析
- 行为结构呈金字塔分布：浏览（pv）占绝对多数，购买（buy）占比极低。
- 时序规律：捕捉到 12 月初的**双十二预热期活跃度升温**；活跃高峰落在**晚 21-22 点**（时区修正后）。

### 2、转化漏斗与行为路径
- 以**用户**为单位构建 `浏览 → 加购 → 购买` 漏斗。
- 重点分析**加购未购买**这一高价值流失群体。
- 对比收藏与加购两种意向信号对最终转化的预测力。

### 3、用户分层
- 基于行为频次，用**分位数法**将用户划分为高/中/低活跃三层，为后续假设检验提供分组依据。

### 4、假设检验（核心）
每个检验均遵循 **业务问题 → 假设 → 方法选择 → 结果 → 业务结论** 五段式：

| 检验 | 业务问题 | 方法 |
|------|----------|------|
| 检验一 | 周末是否提升购买转化率？ | 卡方独立性检验 |
| 检验二 | 晚高峰加购转化是否更高？ | 卡方 / 两比例检验 |
| 检验三 | 高/低活跃用户浏览深度是否有差异？ | T 检验 → 非参数检验 |

---

## 五、核心结论

**漏斗诊断**：浏览 → 加购是最大流失环节，浏览到购买的整体转化率约为 **68%**。

**高价值人群**：加购未购买用户占加购用户的 **28%**，是定向召回（优惠券、库存提醒）的核心对象。

**经统计验证的运营建议**：
   - 周末与工作日购买转化经卡方检验，差异显著；实际转化率差距为 **4.18%**，据此判断值得加投。
   - 晚高峰加购转化经卡方检验显著高于其他时段，建议在 21-22 点集中流量与推荐资源。
   - 高活跃与低活跃用户浏览深度差异显著（Mann-Whitney U，p = **0.011604**），可据此优化推荐策略。

## 六、项目亮点

- ✅ **时区陷阱的识别与修正**：主动发现并修正了时间戳 UTC 偏差，避免了错误的活跃时段结论。
- ✅ **检验方法的严谨选择**：验证统计假设前提，在数据不满足正态性时改用非参数检验，而非机械套用 T 检验。
- ✅ **业务与统计的结合**：每个检验都从真实业务问题出发，并以"显著性 + 效应大小"双重视角解读结果。

---

## 七、项目局限性

- 数据仅覆盖约 9 天，无法分析长周期留存与复购。
- 数据集不含交易金额，故用户分层基于行为维度而非消费维度（RFM 的金额维度缺失）。
- 假设检验基于自然分组（如周末/工作日），属**准实验（quasi-experiment）**，可能存在混杂因素（confounding），结论的因果强度弱于随机对照实验。
